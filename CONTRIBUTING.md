# Contributing Guidelines

## Code Style
- Use **PEP8** for Python files.
- Follow the language‑specific style guide for other languages.
- Run linting before opening a pull request.

## Pull Request Process
- Fork the repository and create a feature branch.
- Ensure all tests pass and the code builds.
- Provide a clear description of the changes.
- Link any related issue using `Closes #<issue-number>`.
- Request at least one review before merging.

## Issues
- Search existing issues before opening a new one.
- Provide a minimal reproducible example.
- Tag with appropriate labels (e.g., `enhancement`, `documentation`).
